# Visualize Eval

Ruthless quality evaluation for HTML visualizations. The bar is: **would a professional designer ship this?**

## The Quality Bar

**We are not grading on a curve.** We are comparing against:
- A senior designer at Stripe making a presentation
- A data visualization at the New York Times
- A Gamma Pro template with custom branding
- An Apple keynote slide

If our output doesn't compete, it fails.

## Evaluation Protocol

For EVERY visualization, execute ALL steps. No shortcuts.

### Step 1: Browser Testing (Automated)

Open the HTML file in a real browser. Test at three viewports:

```
Desktop:  1280 × 720  (presentation standard)
Tablet:   768 × 1024  (iPad)
Mobile:   375 × 667   (iPhone SE)
```

For each viewport, take a screenshot of:
- First view / hero
- 2-3 key content areas
- Last section / footer

Additionally:
- Open the hamburger menu → screenshot
- Toggle to light theme → screenshot first view
- Toggle to dark theme → screenshot first view

### Step 2: Automated Technical Audit

Run this check script in the browser. Every `false` is a deduction.

```javascript
(function() {
  const audit = {};
  const style = [...document.querySelectorAll('style')].map(s => s.textContent).join(' ');
  const html = document.documentElement.outerHTML;

  // === STRUCTURE ===
  audit.hasDoctype = html.startsWith('<!DOCTYPE html>') || html.startsWith('<!doctype html>');
  audit.hasLangAttr = !!document.documentElement.lang;
  audit.hasCharset = !!document.querySelector('meta[charset]');
  audit.hasViewport = !!document.querySelector('meta[name="viewport"]');
  audit.hasTitle = document.title.length > 0;
  audit.titleNotDefault = !document.title.match(/^(untitled|document|visualization)$/i);

  // === MENU ===
  audit.menuExists = !!document.querySelector('.viz-menu');
  audit.menuHasToggle = !!document.querySelector('.viz-menu-toggle');
  audit.menuHasDropdown = !!document.querySelector('.viz-menu-dropdown');
  audit.menuHasThemeBtn = !!html.match(/cycleTheme|themeLabel|theme.*toggle/i);
  audit.menuHasDownload = !!html.match(/downloadImage|html-to-image|htmlToImage/i);
  audit.menuHasPrint = !!html.match(/window\.print|printPDF/i);

  // === THEME SYSTEM ===
  audit.hasCSSVars = !!style.match(/--bg\s*:/);
  audit.hasDarkTheme = !!style.match(/\.theme-dark|:root/);
  audit.hasLightTheme = !!style.match(/\.theme-light/);
  audit.hasAutoTheme = !!style.match(/prefers-color-scheme/);
  audit.themeLocalStorage = !!html.match(/localStorage.*theme/i);

  // === TYPOGRAPHY ===
  audit.hasInterFont = !!html.match(/fonts\.googleapis.*Inter|font-family.*Inter/);
  audit.hasFontStack = !!style.match(/(-apple-system|BlinkMacSystemFont|system-ui)/);
  audit.bodyFontSize = parseFloat(getComputedStyle(document.body).fontSize);
  audit.bodyFontSizeOK = audit.bodyFontSize >= 14;

  // === LAYOUT ===
  audit.usesFlexbox = !!style.match(/display\s*:\s*flex/);
  audit.usesGrid = !!style.match(/display\s*:\s*grid/);
  audit.hasMaxWidth = !!style.match(/max-width/);

  // === PRINT ===
  audit.hasPrintStyles = !!style.match(/@media\s*print/);
  audit.printHidesMenu = !!style.match(/@media\s*print[\s\S]*?\.viz-menu[\s\S]*?display\s*:\s*none/);
  audit.hasPrintColorAdjust = !!style.match(/print-color-adjust/);

  // === ANIMATIONS ===
  audit.hasKeyframes = !!style.match(/@keyframes/);
  audit.hasTransitions = !!style.match(/transition\s*:/);
  audit.hasReducedMotion = !!style.match(/prefers-reduced-motion/);

  // === ACCESSIBILITY ===
  audit.hasAriaLabels = !!html.match(/aria-label/);
  const contrastOK = true; // Would need actual color checking
  audit.semanticHTML = !!html.match(/<(header|main|nav|section|article|footer)/);

  // === PERFORMANCE ===
  audit.fileSizeKB = Math.round(new Blob([html]).size / 1024);
  audit.fileSizeOK = audit.fileSizeKB < 200; // Under 200KB
  audit.noExternalImages = document.querySelectorAll('img[src^="http"]').length === 0;
  audit.inlineSVGCount = document.querySelectorAll('svg').length;

  // === CDN DEPS ===
  audit.htmlToImageLoaded = typeof htmlToImage !== 'undefined';
  const scripts = [...document.querySelectorAll('script[src]')];
  audit.cdnLibraries = scripts.map(s => {
    const url = new URL(s.src);
    return url.pathname.split('/npm/')[1]?.split('/')[0] || url.hostname;
  });

  // === SCORE TECHNICAL ===
  const bools = Object.entries(audit).filter(([k,v]) => typeof v === 'boolean');
  const passed = bools.filter(([k,v]) => v).length;
  const total = bools.length;
  audit._technicalScore = `${passed}/${total} (${Math.round(passed/total*100)}%)`;
  audit._failures = bools.filter(([k,v]) => !v).map(([k]) => k);

  console.table(audit);
  return audit;
})();
```

**Technical score must be ≥ 90%.** Every failure needs a justification or a fix.

### Step 3: Visual Inspection (8 Dimensions)

Score each dimension 1-10. Be HARSH. A 7 means "acceptable but forgettable." An 8 means "good." A 9 means "impressive." A 10 is "I want to screenshot and share this."

#### D1: First Impression (weight: 15%)
The 2-second test. Open the file and react.
- Does it look professional INSTANTLY?
- Is there a clear focal point?
- Does the color palette feel intentional?
- Would you be proud to show this in a meeting?
- **10:** "Wow, that's beautiful" — involuntary positive reaction
- **8:** "That looks professional" — clean and polished
- **6:** "That's fine" — generic, forgettable
- **4:** "That looks like a template" — clearly AI-generated feeling
- **2:** "That looks broken" — layout issues, ugly colors

#### D2: Typography & Hierarchy (weight: 15%)
- Is there a clear visual hierarchy? (can you tell what to read first, second, third?)
- Are font sizes appropriate? (headlines big, body comfortable, labels small)
- Is line-height comfortable? Not cramped, not too loose?
- Max 2 font families?
- Do headings use weight AND size for hierarchy, not just bold?
- **Deductions:** All same size (-3), cramped text (-2), no clear reading order (-3), tiny fonts (-2)

#### D3: Color & Contrast (weight: 10%)
- Is the palette harmonious? (2-3 colors + neutrals)
- Does every text pass WCAG AA contrast (4.5:1)?
- Are accent colors used purposefully (to draw attention to key info)?
- Does light theme look as good as dark theme?
- Are positive/negative values color-coded correctly?
- **Deductions:** Clashing colors (-3), low contrast (-3), too many colors (-2), light theme broken (-2)

#### D4: Layout & Spacing (weight: 15%)
- Is spacing consistent? (same gaps between similar elements)
- Is there generous whitespace? (content never touches edges)
- Is the layout responsive? (doesn't break at 768px, 375px)
- Are elements properly aligned? (nothing looks "off")
- Does the layout guide the eye naturally?
- **Deductions:** Cramped (-3), inconsistent spacing (-2), broken at mobile (-3), misalignment (-2)

#### D5: Content Quality (weight: 15%)
- Is the information accurate and clear?
- Can you understand the main message in 5 seconds?
- Is there the RIGHT amount of content? (not too sparse, not overwhelming)
- Are data/charts labeled clearly?
- Zero placeholder content?
- **Deductions:** Placeholder text (-5), unclear message (-3), too much text (-2), wrong chart type (-2)

#### D6: Interactivity & UX (weight: 10%)
- Does the hamburger menu work? (open, close, outside click, Escape)
- Does theme toggle work and persist?
- Does download-as-PNG work?
- Does print produce a clean PDF?
- (Slides) Do keyboard, click, and swipe navigation all work?
- Are hover states present on interactive elements?
- **Deductions:** Menu missing (-4), broken nav (-3), no hover states (-1), download broken (-2)

#### D7: Technical Excellence (weight: 10%)
- Zero console errors?
- File size reasonable (<200KB)?
- No external image dependencies?
- Proper HTML semantics?
- Animations smooth (no jank)?
- Print styles complete?
- `prefers-reduced-motion` respected?
- **Deductions:** Console errors (-2 each), external images (-2), no print styles (-2), janky animations (-1)

#### D8: Shareability / Wow Factor (weight: 10%)
- Would you screenshot this and post it on Twitter?
- Would someone ask "how did you make this?"
- Does it look BETTER than what you'd get from Gamma/Canva?
- Is there at least one "delightful" detail? (animation, clever layout, beautiful chart)
- Could this be used in a REAL presentation to a REAL audience?
- **10:** People would share this unprompted
- **8:** Noticeably better than standard tools
- **6:** On par with standard tools
- **4:** Worse than what you'd get from Canva
- **2:** Embarrassing

### Step 4: Calculate Overall Score

```
overall = (D1 × 0.15) + (D2 × 0.15) + (D3 × 0.10) + (D4 × 0.15) +
          (D5 × 0.15) + (D6 × 0.10) + (D7 × 0.10) + (D8 × 0.10)
```

### Quality Gates

| Gate | Criteria | Action |
|------|----------|--------|
| **🚀 VIRAL** | Overall ≥ 9.5, all ≥ 9 | Ship immediately. Share everywhere. |
| **✅ SHIP** | Overall ≥ 9.0, all ≥ 8 | Ready for release. |
| **⚠️ GOOD** | Overall ≥ 8.0, all ≥ 7 | Publishable but not impressive. Fix before featuring. |
| **🔧 NEEDS WORK** | Overall ≥ 7.0 or any < 7 | Fix critical issues, re-evaluate. |
| **❌ FAIL** | Overall < 7.0 or any < 5 | Significant rework needed. |

**Our target: SHIP (≥ 9.0) for every test case. VIRAL (≥ 9.5) for showcase examples.**

### Step 5: Generate Fix List

For every dimension scoring below 9:

```markdown
### Fixes Required

#### [CRITICAL] — blocks shipping (score < 8)
1. D6: Menu missing → implement full menu component from references/menu.md

#### [HIGH] — significantly improves quality (score 8)
2. D1: Hero gradient too flat → use 3-stop gradient with subtle mesh overlay
3. D4: Card spacing inconsistent → standardize all gaps to 24px

#### [MEDIUM] — polish (score 8-9)
4. D8: No entrance animations → add staggered fadeInUp on card elements
5. D3: Light theme accent too bright → darken by 10%

#### [LOW] — nice to have (score 9)
6. D2: Could use Inter font instead of system fonts
```

### Step 6: Fix → Re-evaluate → Loop

1. Apply fixes (highest priority first)
2. Re-open file in browser
3. Re-score ONLY affected dimensions
4. Check quality gate
5. If not SHIP, loop back to Step 5

**Maximum 3 fix loops per visualization.** If it can't reach SHIP in 3 loops, the issue is systemic — fix SKILL.md instead.

## Systemic vs Local Fixes

**This is critical.** When you find an issue:

Ask: "Would this same problem appear if I generated a DIFFERENT visualization?"

- **YES → Systemic fix.** Update SKILL.md, references, or templates. This improves ALL future outputs.
- **NO → Local fix.** Fix this specific file only.

**Always prefer systemic fixes.** One SKILL.md change > fixing 40 individual files.

Examples:
- ❌ "This chart is too small" → resize chart (local)
- ✅ "Charts default too small" → add to SKILL.md: "Charts minimum 60% container width" (systemic)

- ❌ "Fonts look generic" → change to Inter (local)
- ✅ "No font specified" → add Inter CDN to SKILL.md output rules (systemic — already done!)
