# Visualize Eval

Score and improve HTML visualizations through systematic evaluation.

## The Loop

```
Generate (visualize skill)
    ↓
Evaluate (this skill) → Score + Fix List
    ↓
Fix (apply changes)
    ↓
Re-evaluate → Score improved?
    ↓
Ship when overall ≥ 8.5, no dimension < 7
```

## How to Evaluate

### Step 1: Open & Screenshot

Open the HTML file in a browser at 1280×720. Take screenshots of:
- First view (hero/title)
- 2-3 key content sections
- Mobile view (375px width)
- Menu open state
- Light AND dark theme

### Step 2: Technical Checks

Run these checks programmatically:

```javascript
// In browser console or via evaluate
(function() {
  const checks = {};

  // 1. Console errors
  checks.consoleErrors = 'Check manually';

  // 2. External dependencies
  const scripts = [...document.querySelectorAll('script[src]')];
  const links = [...document.querySelectorAll('link[href]')];
  checks.externalDeps = scripts.map(s => s.src).concat(links.map(l => l.href));

  // 3. Menu present
  checks.menuPresent = !!document.querySelector('.viz-menu');
  checks.menuToggleWorks = !!document.querySelector('.viz-menu-dropdown');

  // 4. Theme system
  checks.themeVars = getComputedStyle(document.documentElement).getPropertyValue('--bg') !== '';
  checks.lightThemeExists = !!document.querySelector('style').textContent.match(/\.theme-light/);

  // 5. Print styles
  checks.printStyles = !!document.querySelector('style').textContent.match(/@media\s+print/);

  // 6. Responsive
  checks.hasViewportMeta = !!document.querySelector('meta[name="viewport"]');
  checks.usesFlexOrGrid = !!document.querySelector('style').textContent.match(/(display:\s*(flex|grid))/);

  // 7. Accessibility
  checks.hasLangAttr = !!document.documentElement.lang;
  checks.hasTitle = !!document.title;

  // 8. Images/SVG
  const imgs = document.querySelectorAll('img');
  checks.externalImages = [...imgs].filter(i => i.src.startsWith('http')).length;
  checks.inlineSVGs = document.querySelectorAll('svg').length;

  // 9. html-to-image loaded
  checks.htmlToImageLoaded = typeof htmlToImage !== 'undefined';

  // 10. File size
  checks.htmlSize = new Blob([document.documentElement.outerHTML]).size;
  checks.htmlSizeKB = Math.round(checks.htmlSize / 1024);

  console.table(checks);
  return checks;
})();
```

### Step 3: Score Each Dimension

See [../references/eval.md](../references/eval.md) for the full rubric.

Score format:
```
## Evaluation: [filename]

| Dimension | Score | Notes |
|-----------|-------|-------|
| Visual Design | X/10 | ... |
| Content Effectiveness | X/10 | ... |
| Technical Quality | X/10 | ... |
| Interactivity & UX | X/10 | ... |
| Wow Factor | X/10 | ... |
| **Overall** | **X.X/10** | weighted |

### Quality Gate: [SHIP / GOOD / NEEDS WORK / FAIL]
```

### Step 4: Generate Fix List

For each score below 8, produce specific, actionable fixes:

```
### Fixes (priority order)

1. **[CRITICAL]** Menu missing → Add standard menu component from references/menu.md
2. **[HIGH]** Heading font too small (24px) → Increase to 36px+ for slide titles
3. **[MEDIUM]** No entrance animations → Add fadeUp keyframe with staggered delays
4. **[LOW]** Could add subtle gradient to hero background
```

### Step 5: Apply & Re-evaluate

After fixes are applied:
1. Re-open the file (hard refresh)
2. Re-score ONLY the dimensions that were affected
3. Verify overall meets quality gate
4. If not, repeat from Step 4

## Automated Batch Evaluation

For stress-testing multiple outputs at once:

```
For each test case in the stress test suite:
  1. Generate the visualization using the visualize skill
  2. Open in browser, run technical checks
  3. Screenshot key views
  4. Score all 5 dimensions
  5. Log results to eval-results.md
  6. If score < 8.5, generate fixes and re-run
```

## What "Viral Quality" Means

A visualization is viral-ready when someone would:
1. **Screenshot it** and share on Twitter/LinkedIn
2. **Send the HTML file** to a colleague saying "look what AI made"
3. **Use it in a real meeting** without embarrassment
4. **Ask "how did you make this?"** — the output sells the tool
5. **Prefer it** over what they'd get from Gamma/Canva/PowerPoint

The bar is NOT "good for AI-generated." The bar is "good, period."
