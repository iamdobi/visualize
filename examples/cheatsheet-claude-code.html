Claude Code Cheat Sheet

The essential reference for Anthropic's AI coding CLI

Setup & Install
  • npm i -g @anthropic-ai/claude-code Install globally
  • claude Launch in current directory
  • claude -p "query" One-shot prompt mode
  • claude config Open settings
  • claude update Update to latest version
  • cat file | claude -p "fix" Pipe input as context
Key Commands
  • /help Show all slash commands
  • /compact Summarize & compress context
  • /clear Reset conversation history
  • /cost Show token usage & cost
  • /model Switch model mid-session
  • /review Code review current changes
  • /init Create CLAUDE.md for project
  • /memory Edit project memory file
Workflow Tips
  • Start with a plan — ask Claude to outline before coding
  • Use /compact when context gets long
  • Add CLAUDE.md to repos for persistent instructions
  • Give Claude tests to validate against
  • Commit often — Claude can revert if needed
  • Use headless mode for CI/CD pipelines
Run claude -p "..." | pbcopy to pipe output directly to clipboard.
Multi-file Editing
  • Claude reads & writes files across your project
  • Reference files by name — "update App.tsx and its tests"
  • Ask "find all files that..." for codebase search
  • Claude auto-creates new files when needed
  • Uses git diff to track all changes made
  • Respects .gitignore for file discovery
Tool Use Patterns
  • Read Read file contents
  • Write Create/overwrite files
  • Edit Surgical find & replace edits
  • Bash Run shell commands
  • Glob Search for files by pattern
  • Grep Search file contents (regex)
  • LS List directory contents
Claude chains tools automatically — just describe the goal.
Keyboard Shortcuts
  • Ctrl+C Cancel current generation
  • Ctrl+D Exit Claude Code
  • Esc Cancel current input
  • Up Arrow Previous message
  • Tab Accept suggestion
  • Shift+Tab Toggle vim mode
Pro Tips
  • Be specific: "Add error handling to the fetch in api.ts line 42"
  • Paste error messages directly — Claude reads stack traces
  • Use git branches for experimental Claude changes
  • "Write tests first, then implement" — TDD with Claude
  • Add CLAUDE.md to every repo with project conventions
  • Use --allowedTools flag to pre-approve tools
  • Chain sessions: output of one becomes input to next
  • "Show me the plan first" before big refactors
Permissions
  • Read — always allowed (safe)
  • Write/Edit — asks on first use
  • Bash — asks for each new command
  • Accept once = allowed for session
  • --dangerously-skip-permissions for CI
